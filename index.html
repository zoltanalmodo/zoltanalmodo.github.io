- Motivation

/how i started programming

i have a passion for creation and i express it by drawing, painting and design.
during my studies as an industrial designer i learned how to analyze problems and build products to solve them.

i started programming at around 2013 in Html / CSS when i was working on a webshop for a London based furniture shop called OneDeko at Old Spitalfields market.

the shop was runned on Prestashop (a popular free ecommerce CMS platform) which meant i had to modify it here and there using Html and CSS.
i migrated the shop from version to version and had to learn more about Mysql, CSV, fix few bugs on the Theme's CSS, or install some addons by reading the code and following instructions.
this is how i learned more about programming.

later on i started to work from home and had a bit more time this allowed me to attend meetups.

since then i learned a lot, played with coding and finally managed to build my first website.

- Tech interests

/my way here

i had an idea about an application for artists to draw submit and mix drawings with others.

i call it Doodle Disco.
i decided to build it as best as i can. first with a developer friend of mine...
we did an early mockup with some basic functions on an iPhone.
suddenly he had to leave as he had a baby boy and he could no longer work together on the project.
i started to look for developers in London and attended various meetups. with little to no success finding a partner.

i feel what is it like to sudy in a community with like minded people.
also know that to build something requires a team of people so to enter the program would allow me to make friends and participate in common projects would get me more experience.

i feel like i often just need someone to show my code to and it takes just a different approach for things to click together.
this is much faster than i could learn on my own.
i got a lot of guidance and learning material which helped me a lot!

i know i would have to find employment after the course to get more experienced and learn before i can build something more complex.

/in the future

this is one of the ideas i would like to build one day.

a drawing tool + community platform in one which allows anyone to draw with their finger (or mouse) and submit drawings or parts of drawings into a database.
users can share their creations with others on various platforms and devices throughout the internet.

it raises some quiestions about today's artworld:
what is art today? how do we access and view art? who is an artist? what artists are doing? how art shapes society?
and how could art, science, software and internet technology work together to change society?

thought of artists and scientists like Leonardo and Michelangelo they were all learning the newest techniques and concepts of their time.
i imagined when perspective was changing the way of illustrating a 3 dmiensional space in 2 a dimensional surface.

i know that if i want to create something meaningful today, i have to know how to read and write code and develop software.

i would like to learn more about...
how to write server side applications? how to create functions to create visually engaging websites?
how to create collaboration tools and run an application on multiple devices?

how to create server side application for storing and distributing data?
how to handle data input X output from various / multiple digital devices?
how to handle user's data and allow them to make connections with other users and software applications?
